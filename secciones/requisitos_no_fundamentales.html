Requisitos no funcionales

Descripción

Los requisitos no funcionales son aquellos requisitos que debe tener un proyecto pero cuentan con ninguna funcionalidad, no tienen una tarea por realizar en el propio proyecto, esto va mas enfocado a atributos de calidad, seguridad, rendimiento y escalabilidad.
Algunas practicas que se deben tener en cuenta al crear estos requisitos no funcionales, Los requisitos deben ser claros, ser especifico sobre lo que se requiere, utilizar lenguaje sencillo, los requisitos deben ser alcanzables y revisar y modificar los requisitos según sea necesario.

Usabilidad

  1. Seguridad: El sistema debe estar protegido contra accesos no autorizados.
  2. Escalabilidad: El proyecto debe ser bien diseñado para que se pueda expandir en caso de ser necesario
  3. Disponibilidad: El sistema debe estar disponible cuando sea necesario y ademas la información debe ser proporcionada cuando sea necesaria
  4. Mantenimiento: El proyecto debe ser fácil de mantener y realizar actualizaciones
  5. Compatibilidad: El sistema debe cumplir con todas las leyes y reglamentos aplicables
